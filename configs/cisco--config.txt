=== SOLUTION ARCHITECTURE BLUEPRINT ===

Device Models and Roles:
- Spine Switch: Cisco Nexus 9364C - Spine1
- Spine Switch: Cisco Nexus 9364C - Spine2
- Spine Switch: Cisco Nexus 9364C - Spine3
- Spine Switch: Cisco Nexus 9364C - Spine4
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf1
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf2
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf3
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf4
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf5
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf6
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf7
- Leaf Switch: Cisco Nexus 9348GC-FXP - Leaf8
- Border Router: Cisco ASR 1006-X - Border1
- Border Router: Cisco ASR 1006-X - Border2

Network Topology Description:
- Clos spine-leaf architecture with 4-way spine and 8 leafs.
- Each leaf switch dual-homed to all 4 spines using 100G (ECMP).
- All leaf switches perform VTEP (VXLAN Tunnel Endpoint) for multi-tenant VRFs.
- BGP EVPN as control-plane for VXLAN: every leaf and border forms BGP EVPN with spines (route-reflectors).
- OSPF underlay with unique router-ids, spine/leaf loopbacks in /32s, all P2P spine-leaf links in 10.10.255.0/24, /30s.
- VLANs/VNIs mapped to multiple VRFs with address segmentation.
- Anycast gateway for SVI mobility on all active leafs (same MAC/SVI IP).
- Border routers participate in OSPF underlay + eBGP to corporate WAN and Internet.
- Security: Strong AAA, SSH, NetFlow, CoPP, microsegmentation on DB tier, all VRFs with proper ACLs and controls.

=== DEPLOYMENT RECOMMENDATIONS ===

Installation Sequence:
1. Pre-configure management (mgmt0) IPs and AAA/SSH on all devices.
2. Configure spine switches (Spine1-4): loopbacks, OSPF underlay, BGP route-reflection, NVE overlay, security.
3. Implement leaf switch configs (Leaf1-8): loopback, uplink, OSPF underlay, VRFs/VLANs, BGP EVPN, NVE, SVI, security, ACLs.
4. Apply border router configs (Border1-2): underlay connection, OSPF, BGP to leafs, BGP to WAN, NAT, AAA, security ACLs.
5. Validate all P2P IPs, BGP/OSPF, NVE.
6. Copy all running-config to startup-config on all devices.

Verification Procedures:
- Use `show ip ospf neighbor` on all devices for OSPF neighbor adjacency
- Use `show bgp l2vpn evpn summary` on all leafs and spines for EVPN
- Use `show nve peers` for NVE overlay status
- Use `show interface status` to confirm link status
- Use `show vlan brief` and `show vrf detail` for segmentation
- Use `show access-lists` to validate micro-segmentation
- Use `show standby`/`show vrrp` if any L3 gateway redundancy configured
- Validate management access with SSH

=== DEPLOYMENT SUCCESS CRITERIA ===

✓ CONNECTIVITY: All leafs reach all spines. Anycast GW reachable via all leafs. Border routers reach WAN and Internet.
✓ ROUTING: OSPF fully converged. BGP EVPN adjacencies up on all spines/leafs. VXLAN overlay up. Loopbacks advertised everywhere.
